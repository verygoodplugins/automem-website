including their structure, properties, classification taxonomy, and storage representation. For information about how memories relate to each other, see [Relationship Types](/verygoodplugins/automem/3.2-relationship-types). For details on how memories are searched and retrieved, see [Hybrid Search](/verygoodplugins/automem/3.3-hybrid-search).\n\n* * *\n\n## Overview\n\nEvery memory in AutoMem is a structured data object with both required and optional properties. Memories are classified into specific types (Decision, Pattern, Preference, etc.), enriched with metadata, and stored redundantly in both FalkorDB (graph) and Qdrant (vector) databases. The memory model supports temporal validity windows, confidence scoring, and hierarchical tag organization.\n\n* * *\n\n## Memory Schema\n\n### Core Properties\n\nAll memories contain the following properties:\n\nProperty\n\nType\n\nRequired\n\nDescription\n\n`id`\n\nstring (UUID)\n\nYes\n\nUnique identifier, auto-generated if not provided\n\n`content`\n\nstring\n\nYes\n\nThe actual memory text (minimum 1 character)\n\n`type`\n\nstring\n\nYes\n\nClassification from `MEMORY_TYPES` set\n\n`confidence`\n\nfloat\n\nYes\n\nClassification confidence score (0.0-1.0)\n\n`importance`\n\nfloat\n\nYes\n\nUser-specified or derived importance (0.0-1.0, default: 0.5)\n\n`timestamp`\n\nstring (ISO 8601)\n\nYes\n\nWhen the memory was created or occurred\n\n`tags`\n\narray\\[string\\]\n\nNo\n\nHierarchical categorization tags (e.g., `[\"project:automem\", \"decision\"]`)\n\n`tag_prefixes`\n\narray\\[string\\]\n\nNo\n\nPre-computed lowercase tag prefixes for fast filtering\n\n`metadata`\n\nobject\n\nNo\n\nFlexible JSON object for custom fields\n\n`embedding`\n\narray\\[float\\]\n\nNo\n\n768-dimensional vector for semantic search\n\n### Temporal Properties\n\nProperty\n\nType\n\nRequired\n\nDescription\n\n`t_valid`\n\nstring (ISO 8601)\n\nNo\n\nWhen this memory becomes valid (future-dated memories)\n\n`t_invalid`\n\nstring (ISO 8601)\n\nNo\n\nWhen this memory expires or becomes invalid\n\n`updated_at`\n\nstring (ISO 8601)\n\nYes\n\nLast modification timestamp\n\n`last_accessed`\n\nstring (ISO 8601)\n\nYes\n\nLast retrieval or access timestamp\n\n### Enrichment Properties\n\nProperty\n\nType\n\nDescription\n\n`enriched_at`\n\nstring (ISO 8601)\n\nWhen enrichment pipeline processed this memory\n\n`enrichment_attempts`\n\ninteger\n\nNumber of enrichment attempts (max 3)\n\n`summary`\n\nstring\n\nAuto-generated summary (first sentence, max 240 chars)\n\n`entities`\n\nobject\n\nExtracted entities: `tools`, `projects`, `people`, `concepts`, `organizations`\n\n### State Properties\n\nProperty\n\nType\n\nDescription\n\n`archived`\n\nboolean\n\nWhether memory is archived (excluded from search by default)\n\n`relevance_score`\n\nfloat\n\nDynamic score computed by consolidation engine\n\n**Sources:** [app.py118-227](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L118-L227) semantic types. The classification system is defined in `MEMORY_TYPES`:\n\n### Type Definitions\n\nType\n\nDescription\n\nExample Content\n\n`Decision`\n\nStrategic or technical choices, selected options\n\n\"Chose PostgreSQL over MongoDB for ACID compliance\"\n\n`Pattern`\n\nRecurring behaviors, typical approaches, consistent tendencies\n\n\"Usually write integration tests before deploying\"\n\n`Preference`\n\nLikes/dislikes, favorites, personal or team tastes\n\n\"Prefer async/await over callbacks for clarity\"\n\n`Style`\n\nCommunication approach, formatting preferences, tone\n\n\"Write commit messages in imperative mood\"\n\n`Habit`\n\nRegular routines, repeated actions, scheduled activities\n\n\"Run pytest every morning before starting work\"\n\n`Insight`\n\nDiscoveries, learnings, realizations, key findings\n\n\"Realized embedding batching reduces costs by 40%\"\n\n`Context`\n\nSituational background, circumstances, what was happening\n\n\"During database migration sprint in Q3 2024\"\n\n**Note:** `\"Memory\"` was used as an internal fallback type prior to v0.5.0 but is no longer a valid classification. All memories must be classified into one of the seven specific types.\n\n**Sources:** [app.py118-125](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L118-L125) [app.py1000-1028](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L1000-L1028) classification system that balances speed, cost, and accuracy:\n\n1.  **Explicit Type** (preferred): Client provides `type` field in POST request\n2.  **Regex Pattern Matching** (fast, free): Checks content against predefined patterns\n3.  **LLM Fallback** (accurate, ~$0.02/month): Uses GPT-4o-mini when patterns don't match\n\n### Pattern-Based Classification\n\nThe `MemoryClassifier` class defines regex patterns for each memory type:\n\nType\n\nExample Patterns\n\n`Decision`\n\n`r\"decided to\"`, `r\"chose (\\w+) over\"`, `r\"going with\"`, `r\"opted for\"`\n\n`Pattern`\n\n`r\"usually\"`, `r\"typically\"`, `r\"tend to\"`, `r\"often\"`, `r\"consistently\"`\n\n`Preference`\n\n`r\"prefer\"`, `r\"like.*better\"`, `r\"favorite\"`, `r\"rather than\"`\n\n`Style`\n\n`r\"wrote.*in.*style\"`, `r\"communicated\"`, `r\"formatted as\"`\n\n`Habit`\n\n`r\"always\"`, `r\"every time\"`, `r\"daily\"`, `r\"weekly\"`, `r\"routine\"`\n\n`Insight`\n\n`r\"realized\"`, `r\"discovered\"`, `r\"learned that\"`, `r\"figured out\"`\n\n`Context`\n\n`r\"during\"`, `r\"while working on\"`, `r\"in the context of\"`, `r\"when\"`\n\n**Confidence Calculation:**\n\n*   Base confidence: `0.6` for single pattern match\n*   Boost: `+0.1` for each additional pattern match (max `0.95`)\n\n**Sources:** [app.py996-1117](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L996-L1117) [app.py1000-1028](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L1000-L1028)\n\n### LLM Classification\n\nWhen regex patterns don't match (approximately 30% of cases), the system falls back to GPT-4o-mini:\n\n**System Prompt Structure:**\n\n    You are a memory classification system. Classify each memory into exactly ONE of these types:\n    - Decision: Choices made, selected options, what was decided\n    - Pattern: Recurring behaviors, typical approaches, consistent tendencies\n    - Preference: Likes/dislikes, favorites, personal tastes\n    - Style: Communication approach, formatting, tone used\n    - Habit: Regular routines, repeated actions, schedules\n    - Insight: Discoveries, learnings, realizations, key findings\n    - Context: Situational background, what was happening, circumstances\n    \n    Return JSON with: {\"type\": \"<type>\", \"confidence\": <0.0-1.0>}\n    \n\n**Request Format:**\n\n*   Model: `gpt-4o-mini`\n*   Temperature: `0.3` (deterministic)\n*   Max tokens: `50`\n*   Response format: JSON object\n*   Input limit: First 1000 characters of content\n\n**Validation:**\n\n*   Returned type must exist in `MEMORY_TYPES`\n*   Invalid types fall back to `\"Context\"` with confidence `0.5`\n\n**Cost:** ~$0.02/month at 10 memories/day requiring LLM classification\n\n**Sources:** [app.py1079-1114](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L1079-L1114) [app.py1030-1040](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L1030-L1040) windows:\n\n### Validity Properties\n\nProperty\n\nPurpose\n\nUse Case\n\n`t_valid`\n\nEarliest time this memory is valid\n\nFuture-dated reminders, scheduled knowledge\n\n`t_invalid`\n\nLatest time this memory is valid\n\nExpiring credentials, time-bound decisions\n\n### Query Behavior\n\nBy default, `/recall` excludes memories where:\n\n*   `now < t_valid` (not yet valid)\n*   `now >= t_invalid` (expired)\n\n**Override:** Use `time_query` or explicit `start`/`end` parameters to include expired memories.\n\n**Example Scenarios:**\n\n1.  **Future-dated memory:**\n    \n    Not searchable until November 1, 2025.\n    \n2.  **Expiring credential:**\n    \n    Automatically excluded from search after December 31, 2025.\n    \n\n**Sources:** [app.py363-426](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L363-L426) [app.py569-639](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L569-L639)\n\n* * *\n\n## Metadata Structure\n\nThe `metadata` field is a flexible JSON object for storing arbitrary key-value pairs. Common patterns:\n\n### Standard Metadata Fields\n\n### Entity Extraction Results\n\nThe enrichment pipeline populates `metadata.entities`:\n\n### Enrichment Metadata\n\nAutomatically added by enrichment pipeline:\n\nField\n\nType\n\nDescription\n\n`metadata.enriched_at`\n\nstring\n\nISO timestamp of enrichment\n\n`metadata.enrichment_attempts`\n\ninteger\n\nRetry count (max 3)\n\n`metadata.summary`\n\nstring\n\nAuto-generated summary\n\n`metadata.entities`\n\nobject\n\nExtracted entities by category\n\n### Reserved Fields\n\nThe following fields are **reserved** and should not be placed in `metadata` (they are top-level properties):\n\n*   `type`, `confidence`, `content`, `timestamp`, `tags`, `tag_prefixes`\n*   `importance`, `embedding`, `id`, `archived`, `relevance_score`\n*   `t_valid`, `t_invalid`, `updated_at`, `last_accessed`\n\n**Sources:** [app.py1217-1290](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L1217-L1290) [app.py1292-1300](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L1292-L1300) [scripts/recover\\_from\\_qdrant.py1-150](https://github.com/verygoodplugins/automem/blob/c2a4dae3/scripts/recover_from_qdrant.py#L1-L150)\n\n* * *\n\n## Storage Representation\n\n### FalkorDB (Graph Database)\n\nMemories are stored as nodes with the `Memory` label. Node properties directly map to memory schema:\n\n**Graph Features:**\n\n*   Relationships connect memories (see [Relationship Types](/verygoodplugins/automem/3.2-relationship-types))\n*   Cypher queries enable graph traversal\n*   Indexes on `tags`, `tag_prefixes`, `timestamp`, `importance`\n\n### Qdrant (Vector Database)\n\nMemories are stored as points with payloads mirroring FalkorDB properties:\n\n**Qdrant Features:**\n\n*   Keyword indexes on `tags` and `tag_prefixes` for fast filtering\n*   HNSW index on vectors for approximate nearest neighbor search\n*   Payload filters support complex boolean queries\n\n**Dual Storage Strategy:**\n\n*   FalkorDB is the **source of truth** (required)\n*   Qdrant is **optional** but enables semantic search\n*   Data is written to both databases synchronously\n*   Qdrant can rebuild FalkorDB if graph data is lost\n\n**Sources:** [app.py1770-1900](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L1770-L1900) [app.py1857-1880](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L1857-L1880)  **Created** (`POST /memory`):\n    \n    *   Memory written to FalkorDB\n    *   Embedding queued for generation\n    *   Enrichment job queued\n    *   Returns `201 Created` immediately\n2.  **Enrichment Queued**:\n    \n    *   Job added to in-memory queue\n    *   Worker thread polls queue every 2 seconds\n    *   Max 3 enrichment attempts with exponential backoff\n3.  **Enriching**:\n    \n    *   Entity extraction (spaCy NER + regex)\n    *   Temporal links created (`OCCURRED_BEFORE`)\n    *   Semantic neighbors found (`SIMILAR_TO`)\n    *   Pattern detection (`EXEMPLIFIES`)\n    *   Summary generation (optional)\n4.  **Active**:\n    \n    *   Fully searchable via `/recall`\n    *   Included in graph queries\n    *   Participates in consolidation\n5.  **Decaying**:\n    \n    *   `relevance_score` decreases exponentially\n    *   Decay rate: hourly consolidation task\n    *   Formula: `relevance = base_score * exp(-decay_rate * age)`\n6.  **Archived**:\n    \n    *   Marked `archived: true`\n    *   Excluded from search by default\n    *   Can be restored or deleted\n7.  **Deleted**:\n    \n    *   Removed from both FalkorDB and Qdrant\n    *   Relationships cleaned up\n    *   Irreversible operation\n\n**Sources:** [app.py1292-1450](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L1292-L1450) *\n\n## Validation Rules\n\n### Content Validation\n\n*   **Minimum length:** 1 character (empty strings rejected)\n*   **Maximum length:** No hard limit (practical limit ~10KB for good embedding quality)\n*   **Encoding:** UTF-8\n\n### Type Validation\n\n*   Must be one of: `Decision`, `Pattern`, `Preference`, `Style`, `Habit`, `Insight`, `Context`\n*   Case-sensitive\n*   Invalid types return `400 Bad Request` with valid options listed\n\n### Confidence Validation\n\n*   Range: `0.0` to `1.0` (inclusive)\n*   Default: `0.9` if type explicitly provided, computed otherwise\n*   Non-numeric values rejected\n\n### Importance Validation\n\n*   Range: `0.0` to `1.0` (inclusive)\n*   Default: `0.5`\n*   Used in search ranking and consolidation\n\n### Tag Validation\n\n*   Array of strings or comma-separated string\n*   Normalized to lowercase\n*   Hierarchical prefixes auto-computed (e.g., `\"project:automem:api\"` → `[\"project\", \"project:automem\", \"project:automem:api\"]`)\n*   Empty tags filtered out\n\n### Timestamp Validation\n\n*   Must be ISO 8601 format\n*   Automatically converted to UTC with `+00:00` timezone\n*   Strings ending in `Z` converted to `+00:00`\n*   Invalid timestamps return `400 Bad Request`\n\n### Embedding Validation\n\n*   Must be exactly 768 dimensions\n*   All values must be numeric\n*   Auto-generated if omitted (OpenAI `text-embedding-3-small` or deterministic placeholder)\n\n**Sources:** [app.py213-361](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L213-L361) For details on how relationships influence recall scoring, see [Hybrid Search](/verygoodplugins/automem/3.3-hybrid-search).\n\n* * *\n\n## Overview\n\nAutoMem implements **11 distinct relationship types** that connect Memory nodes in the graph database. Unlike traditional vector databases that only support similarity-based retrieval, typed relationships enable:\n\n*   **Causal reasoning** - Understanding why decisions were made and what they led to\n*   **Temporal sequencing** - Tracking how knowledge evolved over time\n*   **Preference modeling** - Recording what approaches are preferred and why\n*   **Pattern reinforcement** - Connecting examples to recurring patterns\n*   **Conflict resolution** - Marking outdated or contradictory information\n*   **Hierarchical organization** - Structuring memories into logical groupings\n\nEach relationship type carries semantic meaning that influences graph traversal, recall scoring, and consolidation behavior.\n\n**Sources:** Decision→Outcome\n\n`OCCURRED_BEFORE`\n\nTemporal precedence\n\nDirectional\n\nEvent sequencing, chronological ordering\n\n### Enhanced PKG Relationship Types\n\nEight additional types inspired by outdated approaches\n\n`REINFORCES`\n\nSupporting evidence\n\nDirectional\n\n`strength`, `observations`\n\nStrengthening patterns with Query text is converted to a vector using the configured embedding provider, then a cosine similarity search retrieves the top candidates.\n\nThe `_vector_search()` function handles both explicit embeddings (passed by client) and on-demand generation. If Qdrant is unavailable, the system gracefully degrades to graph-only mode.\n\n**Key behaviors:**\n\n*   Returns empty list if no query text and no explicit embedding provided\n*   Applies tag filters via `_build_qdrant_tag_filter()`\n*   Deduplicates results using `seen_ids` set\n*   Attaches relations by calling `_fetch_relations()` for each hit\n\n**Sources:** [app.py612-683](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L612-L683) [automem/stores/vector\\_store.py](https://github.com/verygoodplugins/automem/blob/c2a4dae3/automem/stores/vector_store.py#LNaN-LNaN) [automem/config.py128](https://github.com/verygoodplugins/automem/blob/c2a4dae3/automem/config.py#L128-L128)\n\n* * *\n\n### Keyword Search (Lexical Matching)\n\nKeyword search performs text matching in FalkorDB's graph store using Cypher queries. Inspiration\n\nThe consolidation engine implements memory principles from neuroscience research:\n\nTask\n\nBiological Analog\n\nImplementation\n\n**Decay**\n\nSynaptic weakening over time\n\nExponential relevance decay based on age and access\n\n**Creative**\n\nREM sleep association formation\n\nRandom memory sampling + similarity analysis\n\n**Cluster**\n\nMemory compression during sleep\n\nSemantic grouping + meta-pattern creation\n\n**Forget**\n\nControlled forgetting during consolidation\n\nArchival before deletion, importance-weighted\n\n**Key Research Connections:**\n\n*   **Exponential decay** mirrors [forgetting curves](https://en.wikipedia.org/wiki/Forgetting_curve) (Ebbinghaus, 1885)\n*   **REM-like processing** inspired by [memory replay during sleep](https://www.ncbi.nlm.nih.gov/pmc/articles/PMC4648295/)\n*   **Clustering** implements principles from [MELODI](https://arxiv.org/html/2410.03156v1) (DeepMind, 2024) for memory compression\n*   **Controlled forgetting** follows [active forgetting research](https://www.nature.com/articles/nrn.2016.147) showing forgetting aids learning\n\n**Sources:**
