bridge layer that enables cloud-based AI platforms to access AutoMem's memory system. The bridge translates MCP protocol messages to AutoMem's HTTP API, allowing platforms like ChatGPT, Claude.ai, Claude Mobile, and ElevenLabs to store and recall memories without requiring local installation.\n\nFor direct API usage without MCP, see [Direct API Usage](/verygoodplugins/automem/6.2-direct-api-usage). For local development tools (Cursor, Claude Desktop, Claude Code), use the `@verygoodplugins/mcp-automem` npm package instead of this bridge. For authentication details, see [Authentication](/verygoodplugins/automem/6.3-authentication). For voice-specific Alexa integration details, see [Alexa Integration](/verygoodplugins/automem/6.4-alexa-integration).\n\n* * *\n\n## When to Use the MCP Bridge\n\nThe MCP bridge is required when AI platforms cannot connect directly to local services. The decision matrix:\n\nClient Type\n\nUse MCP Bridge?\n\nRationale\n\nChatGPT (web/mobile)\n\n✅ Yes\n\nCloud-based, requires HTTPS MCP endpoint\n\nClaude.ai (web)\n\n✅ Yes\n\nCloud-based, requires package\n\nDirect HTTP clients\n\n❌ No\n\nUse AutoMem API directly\n\n**Sources:** [docs/MCP\\_SSE.md18-83](https://github.com/verygoodplugins/automem/blob/c2a4dae3/docs/MCP_SSE.md#L18-L83) authentication methods with extraction priority order.\n\n**Token Extraction Priority:**\n\nThe [\\`getAuthToken()\\`](https://github.com/verygoodplugins/automem/blob/c2a4dae3/`getAuthToken\\(\\)`) function at [mcp-sse-server/server.js507-515](https://github.com/verygoodplugins/automem/blob/c2a4dae3/mcp-sse-server/server.js#L507-L515) implements this order:\n\n1.  `Authorization: Bearer {token}` header (preferred)\n2.  `X-API-Key: {token}` header\n3.  `X-API-Token: {token}` header\n4.  `?api_key={token}` query parameter\n5.  `?apiKey={token}` query parameter\n6.  `?api_token={token}` query parameter\n\n**Environment Variable Configuration:**\n\nVariable\n\nPurpose\n\nUsed By\n\n`AUTOMEM_API_TOKEN`\n\nDefault token for API authentication\n\nAll endpoints when no token provided\n\n`AUTOMEM_API_URL`\n\nAutoMem API base URL\n\nAutoMemClient initialization\n\n`AUTOMEM_ENDPOINT`\n\nLegacy alias for `AUTOMEM_API_URL`\n\nFallback support\n\n**URL Token Security Note:**\n\nQuery parameter tokens appear in:\n\n*   Server logs: `[AutoMem] GET http://...?api_key=xxx`\n*   Proxy logs (if using reverse proxy)\n*   Browser history (for web clients)\n\nRecommendation: Use header-based authentication (`Authorization: Bearer`) when supported by the platform.\n\n**Sources:**
