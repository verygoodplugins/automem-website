Release Flow\n\n**Diagram: Typical Release Timeline**\n\n**Day 1**: First `feat:` commit to `main` → release-please creates Release PR with version bump\n\n**Days 2-7**: Additional commits → release-please updates Release PR (version may change if higher-priority commits added)\n\n**Day 8**: Maintainer merges Release PR → Parallel jobs publish to npm and GitHub Releases within ~5
