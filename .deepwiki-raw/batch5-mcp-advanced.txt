[assets/icon.svg](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/assets/icon.svg)\n\nThis document describes the Warp Terminal integration for AutoMem, including its historical implementation and current status.\n\n**Important**: Official Warp Terminal integration was **removed in v0.6.0**. This page documents the historical implementation and provides guidance for manual setup if needed.\n\nFor other terminal-based AI platforms, see [Claude Code](/verygoodplugins/mcp-automem/3.3-claude-code) or [OpenAI Codex](/verygoodplugins/mcp-automem/3.4-openai-codex).\n\n* * *\n\n## Overview\n\nWarp Terminal is an AI-powered terminal that supports Model Context Protocol (MCP) servers for extending AI assistant capabilities. The AutoMem integration provided persistent memory for Warp's AI features, enabling context-aware terminal assistance across sessions.\n\n### Historical Context\n\nThe Warp Terminal integration was:\n\n*   **Added**: v0.4.0 (October 2025)\n*   **Removed**: v0.6.0 (October 2025)\n*   **Reason**: Classified as \"niche use case\" with maintenance burden outweighing usage\n\nThe integration existed for approximately two weeks in the v0.4.x and v0.5.x release series before being deprecated in favor of focusing development on more widely-used platforms (Claude Desktop, Cursor IDE, Claude Code, Codex).\n\n**Sources**: [CHANGELOG.md211-212](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/CHANGELOG.md#L211-L212) [CHANGELOG.md472-486](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/CHANGELOG.md#L472-L486)\n\n* * *\n\n## Architecture (Historical)\n\n### Integration Approach\n\n**Diagram**: Historical Warp Terminal integration architecture\n\nWarp Terminal's AI assistant would use MCP to communicate with the AutoMem server, guided by memory-first rules in `warp-rules.md`. The integration followed the same MCP pattern as other platforms but was optimized for terminal-based workflows.\n\n**Sources**: [CHANGELOG.md472-486](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/CHANGELOG.md#L472-L486)\n\n* * *\n\n## Historical Implementation (v0.4.0 - v0.6.0)\n\n### Template Structure\n\nThe integration consisted of three template files in `templates/warp/`:\n\nFile\n\nPurpose\n\nStatus\n\n`warp-rules.md`\n\nMemory-first terminal assistance rules\n\nRemoved v0.6.0\n\n`mcp.json`\n\nWarp MCP server configuration\n\nRemoved v0.6.0\n\n`README.md`\n\nWarp-specific setup guide\n\nRemoved v0.6.0\n\n### Key Features\n\nBased on the v0.4.0 release notes, the Warp integration provided:\n\n1.  **Project Context Auto-Detection**\n    \n    *   Detected project from `package.json`, `.git/config`, or directory name\n    *   Automatically tagged memories with project context\n    *   Enabled project-scoped memory recall\n2.  **Smart Memory Recall**\n    \n    *   Automatic recall on directory changes (cd commands)\n    *   Context-aware recall for setup/configuration queries\n    *   Time-scoped queries for recent terminal activities\n3.  **Terminal-Optimized Communication**\n    \n    *   Terse, command-first response style\n    *   Focused on actionable commands over explanations\n    *   Shell-friendly output formatting\n4.  **Memory Storage Patterns**\n    \n    *   Captured setup commands and configurations\n    *   Stored debugging patterns and solutions\n    *   Recorded deployment procedures and scripts\n5.  **Cross-Platform Sync**\n    \n    *   Memories stored in Warp accessible from Cursor, Claude Code, Claude Desktop\n    *   Shared project context across development environments\n    *   Unified memory namespace with platform-specific tags\n\n**Sources**: [CHANGELOG.md472-486](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/CHANGELOG.md#L472-L486)\n\n* * *\n\n## Current Status\n\n### Why Was It Removed?\n\nThe integration was removed in v0.6.0 as part of a broader simplification effort:\n\n**Diagram**: Warp integration removal rationale (v0.6.0)\n\nThe removal was part of a philosophy shift toward **\"simplicity and reliability\"** - focusing on production-ready platforms (Claude Desktop, Cursor) while marking experimental features clearly (Claude Code automation).\n\n**Sources**: [CHANGELOG.md391-392](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/CHANGELOG.md#L391-L392) [CHANGELOG.md211-212](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/CHANGELOG.md#L211-L212)\n\n* * *\n\n## Manual Setup (Alternative Approach)\n\nWhile official templates no longer exist, users can manually configure Warp Terminal to use AutoMem via MCP.\n\n### Configuration Steps\n\n1.  **Create Warp MCP Configuration**\n    \n    Warp stores MCP server configurations in its settings. The configuration format follows standard MCP patterns:\n    \n2.  **Add Memory Rules to Warp AI**\n    \n    Create terminal-focused memory rules for Warp's AI assistant:\n    \n    *   Recall memories at conversation start\n    *   Tag memories with `warp` and project context\n    *   Store setup commands with high importance\n    *   Use terminal-optimized query patterns\n3.  **Project Context Detection**\n    \n    Implement manual project detection in your workflow:\n    \n    *   Use `pwd` and repository info to determine project\n    *   Include project name in memory tags\n    *   Query memories with project-specific filters\n\n### Limitations of Manual Setup\n\n*   No automatic template installation\n*   No built-in project auto-detection\n*   Requires manual rule definition\n*   No official documentation or support\n\n**Sources**: [INSTALLATION.md204-236](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/INSTALLATION.md#L204-L236) (Claude Desktop config as reference)\n\n* * *\n\n## Comparison: Terminal-Based Integrations\n\nFeature\n\nWarp (Historical)\n\nClaude Code\n\nOpenAI Codex\n\n**Integration Type**\n\nMCP Rules\n\nMCP + Hooks\n\nMCP + Rules\n\n**Template System**\n\n✅ (removed)\n\n✅ Active\n\n✅ Active\n\n**CLI Installer**\n\n❌\n\n✅ `claude-code`\n\n✅ `codex`\n\n**Auto-detection**\n\n✅ Project context\n\n❌ Manual\n\n❌ Manual\n\n**Status**\n\n🔴 Removed\n\n⚠️ Experimental\n\n✅ Stable\n\n**Recommendation**\n\nUse alternatives\n\nDevelopment use\n\nProduction ready\n\n**Diagram**: Terminal platform comparison\n\nFor terminal-based AI memory, consider:\n\n*   **Claude Code** - Best for terminal development workflows ([Claude Code Setup](/verygoodplugins/mcp-automem/3.3-claude-code))\n*   **OpenAI Codex** - Best for CLI/IDE/cloud integration ([Codex Setup](/verygoodplugins/mcp-automem/3.4-openai-codex))\n*   **Manual Warp Setup** - Only if Warp Terminal is required\n\n**Sources**: [CHANGELOG.md472-486](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/CHANGELOG.md#L472-L486) [README.md48-55](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/README.md#L48-L55)\n\n* * *\n\n## Migration Guidance\n\nUsers who configured Warp Terminal in v0.4.x or v0.5.x should migrate to maintained platforms:\n\n### Migration Paths\n\n**Diagram**: Migration options from Warp integration\n\n### Steps\n\n1.  **Backup existing memories** (if any stored with `warp` tags)\n2.  **Choose replacement platform** based on workflow needs\n3.  **Follow platform-specific setup** in [Installation Guide](/verygoodplugins/mcp-automem/2-getting-started)\n4.  **Update memory tags** to reflect new platform (optional)\n\nMemories stored during Warp usage remain accessible from any MCP-enabled platform - only the client integration changes.\n\n**Sources**: [INSTALLATION.md1-21](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/INSTALLATION.md#L1-L21)\n\n* * *\n\n## Technical Notes\n\n### Why MCP Not Direct API?\n\nUnlike OpenClaw (which uses direct HTTP calls via `curl`), Warp Terminal integration used MCP because:\n\n1.  **Native MCP Support**: Warp Terminal has built-in MCP client capabilities\n2.  **Standard Protocol**: MCP provides consistent interface across platforms\n3.  **Tool Discovery**: MCP allows Warp's AI to discover available memory operations\n\nThe direct API approach (OpenClaw pattern) could also work but would require custom implementation in Warp's AI configuration.\n\n**Sources**: [templates/openclaw/OPENCLAW\\_SETUP.md](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/templates/openclaw/OPENCLAW_SETUP.md) (OpenClaw comparison reference - file not shown but referenced in INSTALLATION.md)\n\n### Historical vs Current MCP Tools\n\nThe MCP tools available to Warp were identical to other platforms:\n\n*   `store_memory`\n*   `recall_memory`\n*   `associate_memories`\n*   `update_memory`\n*   `delete_memory`\n*   `check_database_health`\n\nAll tools remain available via manual MCP configuration.\n\n**Sources**: [INSTALLATION.md949-1155](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/INSTALLATION.md#L949-L1155)\n\n* * *\n\n## Summary\n\n**Warp Terminal Integration Status**:\n\n*   ❌ Official support removed in v0.6.0\n*   📦 Templates removed from codebase\n*   🔧 Manual MCP configuration possible but unsupported\n*   🔄 Migrate to Claude Code or OpenAI Codex for terminal workflows\n\n**Key Takeaway**: The removal of Warp integration reflects AutoMem's focus on widely-adopted platforms where development effort yields broader impact. Terminal-based memory workflows are still supported via Claude Code and OpenAI Codex.\n\n**Sources**: [CHANGELOG.md211-212](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/CHANGELOG.md#L211-L212) [CHANGELOG.md391-392](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/CHANGELOG.md#L391-L392)\n\nDismiss\n\nRefresh this wiki\n\nEnter email to refresh\n\n### On this page\n\n*   [Warp Terminal](#warp-terminal)\n*   [Overview](#overview)\n*   [Historical Context](#historical-context)\n*   [Architecture (Historical)](#architecture-historical)\n*   [Integration Approach](#integration-approach)\n*   [Historical Implementation (v0.4.0 - v0.6.0)](#historical-implementation-v040---v060)\n*   [Template Structure](#template-structure)\n*   [Key Features](#key-features)\n*   [Current Status](#current-status)\n*   [Why Was It Removed?](#why-was-it-removed)\n*   [Manual Setup (Alternative Approach)](#manual-setup-alternative-approach)\n*   [Configuration Steps](#configuration-steps)\n*   [Limitations of Manual Setup](#limitations-of-manual-setup)\n*   [Comparison: Terminal-Based Integrations](#comparison-terminal-based-integrations)\n*   [Migration Guidance](#migration-guidance)\n*   [Migration Paths](#migration-paths)\n*   [Steps](#steps)\n*   [Technical Notes](#technical-notes)\n*   [Why MCP Not Direct API?](#why-mcp-not-direct-api)\n*   [Historical vs Current MCP Tools](#historical-vs-current-"

Platforms)\n\nRelevant source files\n\n*   [INSTALLATION.md](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/INSTALLATION.md)\n*   [README.md](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/README.md)\n*   [assets/icon.svg](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/assets/icon.svg)\n*   [server.json](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/server.json)\n\nThis page documents the **remote MCP sidecar** for connecting cloud-based AI platforms to AutoMem. Cloud platforms like ChatGPT, Claude.ai (web), Claude Mobile (iOS/Android), and ElevenLabs Agents cannot run local MCP servers, so they require an HTTP/SSE bridge.\n\nFor local platform integrations that support stdio-based MCP servers, see:\n\n*   [Claude Desktop](/verygoodplugins/mcp-automem/3.1-claude-desktop)\n*   [Cursor IDE](/verygoodplugins/mcp-automem/3.2-cursor-ide)\n*   [Claude Code](/verygoodplugins/mcp-automem/3.3-claude-code)\n*   [OpenAI Codex](/verygoodplugins/mcp-automem/3.4-openai-codex)\n*   [OpenClaw](/verygoodplugins/mcp-automem/3.5-openclaw)\n*   [Warp Terminal](/verygoodplugins/mcp-automem/3.6-warp-terminal)\n\nFor detailed MCP server implementation (stdio transport), see [MCP Server](/verygoodplugins/mcp-automem/6-mcp-server).\n\n## Overview\n\nThe remote MCP sidecar is a **separate HTTP service** (not part of this repository) that translates HTTP/SSE requests into MCP protocol calls, then forwards them to the AutoMem backend. This enables cloud platforms to use AutoMem memory tools without requiring local process execution.\n\n**Key distinction:**\n\n*   **Local MCP** (pages 3.1-3.6): AI platform → stdio → `src/index.ts` → AutoMem API\n*   **Remote MCP** (this page): AI platform → HTTP/SSE → sidecar → AutoMem API\n\nThe sidecar service is documented and deployed from the [AutoMem service repository](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/AutoMem service repository) not this MCP client repository.\n\nSources: [INSTALLATION.md74-92](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/INSTALLATION.md#L74-L92) [README.md217-232](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/README.md#L217-L232)\n\n## Architecture Comparison\n\n**Transport mechanism differences:**\n\nAspect\n\nLocal MCP (stdio)\n\nRemote MCP (HTTP/SSE)\n\nProcess model\n\nChild process spawned by AI platform\n\nIndependent HTTP service\n\nCommunication\n\nstdin/stdout pipes\n\nHTTP requests / SSE streams\n\nAuthentication\n\nEnvironment variables\n\nHTTP headers or URL params\n\nDeployment\n\nnpm package (`@verygoodplugins/mcp-automem`)\n\nDocker container (separate repo)\n\nState\n\nStateless (new process per session)\n\nSession management via `sessionId`\n\nSources: [INSTALLATION.md74-102](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/INSTALLATION.md#L74-L102) [server.json13-50](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/server.json#L13-L50)\n\n## Supported Platforms\n\nThe remote MCP sidecar enables AutoMem integration for the following cloud platforms:\n\n### ChatGPT (Developer Mode)\n\nChatGPT's Developer Mode supports custom MCP connectors via either Streamable HTTP or SSE. server cannot reach the AutoMem service:\n\n1.  **Verify endpoint**: Check that `AUTOMEM_ENDPOINT` is correct and reachable\n2.  **Test health endpoint**: Run `curl http://your-endpoint/health`\n3.  **Check API key**: Ensure `AUTOMEM_API_KEY` matches your deployed service\n4.  **Network issues**: Verify firewall rules and DNS resolution\n\n**Sources**: [INSTALLATION.md1189-1201](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/INSTALLATION.md#L1189-L1201)\n\n### Configuration Not Found\n\nIf the server cannot find configuration:\n\n1.  **Environment variables**: Ensure `.env` is in the working directory or variables are exported\n2.  **Platform config**: Check that platform config files exist and are readable\n3.  **Resolution priority**: Remember environment variables override platform configs\n\n**Sources**:
