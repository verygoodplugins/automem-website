Add new methods for reusable queries:\n\nAll graph operations should:\n\n*   Use parameterized queries to prevent injection\n*   Handle connection errors gracefully\n*   Return parsed Python objects, not raw result sets\n*   Log errors with context\n\n**Sources:** Referenced from [CLAUDE.md73-76](https://github.com/verygoodplugins/automem/blob/c2a4dae3/CLAUDE.md#L73-L76)\n\n### Adding Background Worker Logic\n\nBackground workers are independent threads managed by `ServiceState`. To add new processing logic:\n\n1.  **Enrichment logic** goes in `automem/enrichment.py` within `EnrichmentWorker._process_job()`\n2.  **Consolidation tasks** go in `automem/consolidation.py` as new methods on `MemoryConsolidator`\n3.  **Embedding logic** goes in `automem/embedding_worker.py` within `EmbeddingWorker._process_batch()`\n\nWorkers use queue-based job management with lock-protected tracking sets to prevent duplicate processing.\n\n**Sources:** Referenced from [CLAUDE.md89-97](https://github.com/verygoodplugins/automem/blob/c2a4dae3/CLAUDE.md#L89-L97)\n\n### Adding a New Embedding Provider\n\nCreate a new provider class implementing the `EmbeddingProvider` interface:\n\nRegister the provider in `automem/embedding/__init__.py` by adding it to the auto-selection logic.\n\n**Sources:** Referenced from [CLAUDE.md131-166](https://github.com/verygoodplugins/automem/blob/c2a4dae3/CLAUDE.md#L131-L166)\n\n## Testing Your Changes\n\nAutoMem uses a multi-tier testing strategy. For complete testing documentation, see [Testing](/verygoodplugins/automem/9.3-testing).\n\n**Quick testing commands:**\n\n**Test files:**\n\n*   `tests/test_app.py` - Core API endpoint tests with `DummyGraph` mock\n*   `tests/test_integration.py` - Full stack tests requiring Docker services\n*   `tests/test_consolidation_engine.py` - Memory consolidation algorithm tests\n*   `tests/test_enrichment.py` - Entity extraction and enrichment tests\n*   `tests/benchmarks/locomo/` - LoCoMo benchmark for long-term memory evaluation\n\n**Sources:** [CLAUDE.md167-197](https://github.com/verygoodplugins/automem/blob/c2a4dae3/CLAUDE.md#L167-L197) [Makefile44-76](https://github.com/verygoodplugins/automem/blob/c2a4dae3/Makefile#L44-L76)\n\n## Code Quality Standards\n\nAutoMem enforces code quality through automated tooling. Run checks before committing:\n\n**Tooling:**\n\nTool\n\nPurpose\n\nConfiguration\n\n`black`\n\nCode formatting (line length, style)\n\nConfigured in `pyproject.toml`\n\n`isort`\n\nImport sorting and organization\n\nConfigured in `pyproject.toml`\n\n`flake8`\n\nLinting (PEP 8, code smells)\n\nConfigured in `.flake8` or `setup.cfg`\n\n**Key conventions:**\n\n*   **Line length**: 88 characters (black default)\n*   **Import order**: stdlib → third-party → local (isort)\n*   **Docstrings**: Google-style for functions and classes\n*   **Type hints**: Encouraged but not required\n*   **Naming**: `snake_case` for functions/variables, `PascalCase` for classes\n\n**Commit message format:** AutoMem uses conventional commits for automated changelog generation:\n\n    feat: add new embedding provider for Ollama\n    fix: resolve race condition in enrichment queue\n    docs: update API reference for recall endpoint\n    test: add integration tests for consolidation\n    \n\n**Sources:** [Makefile53-61](https://github.com/verygoodplugins/automem/blob/c2a4dae3/Makefile#L53-L61) [CLAUDE.md29-31](https://github.com/verygoodplugins/automem/blob/c2a4dae3/CLAUDE.md#L29-L31)\n\n## Deployment Process\n\nAutoMem supports two deployment paths: Railway (recommended for production) and Docker Compose (for self-hosting).\n\n### Railway Deployment\n\nRailway provides one-click deployment with automatic CI/CD:\n\n**Railway deployment flow:**\n\n1.  Push changes to `main` branch on GitHub\n2.  Railway detects changes and triggers build\n3.  Services rebuild and redeploy automatically\n4.  Internal networking via `*.railway.internal` DNS\n5.  Public HTTPS endpoints at `automem.up.railway.app` and `mcp.up.railway.app`\n\n**Critical Railway configuration:**\n\n*   `PORT=8001` must be set (Flask defaults to 5000)\n*   Services bind to `::` for IPv6 dual-stack networking\n*   Shared variables sync `AUTOMEM_API_TOKEN` across services\n*   TCP Proxy required for external FalkorDB access (backups)\n\nFor detailed Railway deployment steps, see [Railway Deployment](/verygoodplugins/automem/7.1-railway-deployment).\n\n### Docker Compose Deployment\n\nFor self-hosting, use Docker Compose:\n\nFor detailed Docker deployment steps, see [Docker Deployment](/verygoodplugins/automem/7.2-docker-deployment).\n\n**Sources:** [CLAUDE.md284-320](https://github.com/verygoodplugins/automem/blob/c2a4dae3/CLAUDE.md#L284-L320) [Makefile86-94](https://github.com/verygoodplugins/automem/blob/c2a4dae3/Makefile#L86-L94)\n\n## Key Development Patterns\n\nUnderstanding these patterns accelerates development and ensures consistency:\n\n### Graceful Degradation\n\n**FalkorDB always succeeds, Qdrant is optional.** All write operations follow this pattern:\n\nThis ensures the system operates in graph-only mode when Qdrant is unavailable.\n\n### Queue-Based Background Processing\n\nWorkers use thread-safe queues with deduplication via tracking sets:\n\nWorkers move IDs from `pending` → `inflight` → removed on completion.\n\n### Authentication Middleware\n\nAll protected endpoints use the `@require_api_token` decorator:\n\nAdmin endpoints require `@require_admin_token` which checks `ADMIN_API_TOKEN`.\n\n### Hybrid Search Scoring\n\nRecall operations combine multiple signals with configurable weights:\n\nWeights are tuned based on LoCoMo benchmark results.\n\n### Time Normalization\n\nAll timestamps use UTC ISO format with automatic parsing:\n\nTemporal queries support relative expressions like \"last week\", \"yesterday\", \"this month\".\n\n**Sources:** [CLAUDE.md322-331](https://github.com/verygoodplugins/automem/blob/c2a4dae3/CLAUDE.md#L322-L331)\n\n## Troubleshooting Common Issues\n\nIssue\n\nSymptom\n\nSolution\n\n`ECONNREFUSED` when MCP Bridge calls API\n\nMCP Bridge can't reach Flask API\n\nSet `PORT=8001` in Flask environment\n\nTests fail with \"Address already in use\"\n\nPort 8001 or 6379 already bound\n\nRun `make clean` to stop Docker services\n\n`DummyGraph` errors in tests\n\nGraph mock not available\n\nSet `PYTEST_DISABLE_PLUGIN_AUTOLOAD=1`\n\nIntegration tests skipped\n\n`AUTOMEM_RUN_INTEGRATION_TESTS` not set\n\nUse `make test-integration` which sets it automatically\n\nRailway deployment fails\n\nVariable resolution errors\n\nCheck shared variables are configured correctly\n\nEmbedding generation fails\n\nNo provider available\n\nSet `OPENAI_API_KEY` or install FastEmbed\n\nFor comprehensive troubleshooting, see [Troubleshooting](/verygoodplugins/automem/8.5-troubleshooting).\n\n**Sources:** [CLAUDE.md167-197](https://github.com/verygoodplugins/automem/blob/c2a4dae3/CLAUDE.md#L167-L197) [Makefile44-76](https://github.com/verygoodplugins/automem/blob/c2a4dae3/Makefile#L44-L76)\n\n## Next
