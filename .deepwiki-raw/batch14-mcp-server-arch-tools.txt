configuration and configuration snippet generation for the mcp-automem package. *\n\n## Summary\n\nThe server architecture implements a clean separation between operational modes:\n\n*   **Server Mode**: Stdio-based MCP protocol server with stderr logging\n*   **CLI Mode**: Command-line utilities that execute and exit\n\nKey architectural decisions:\n\n1.  **Stdio Transport**: All MCP communication occurs over stdin/stdout with logging redirected to stderr\n2.  **Error Resilience**: Guards against broken pipes, tool execution errors wrapped in try-catch\n3.  **Process Management**: Supports stable process titles for safe cleanup by supervisors\n4.  **Configuration**: Environment-based with sensible defaults\n5.  **Static Tool Registry**: Six tools registered at startup with comprehensive schemas\n\nThe implementation at [src/index.ts](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/src/index.ts) serves as both the package entry point and the complete MCP server, making it easy to deploy while maintaining clear separation of concerns through mode detection.\n\n**Sources:** [src/index.ts1-1237](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/src/index.ts#L1-L1237)
