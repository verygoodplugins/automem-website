Acts as the source of truth.\n*   **Qdrant** (optional): Vector database storing 768-dimensional embeddings for semantic similarity search. Enhances performance but is not required for operation.\n\nThe system is designed for **graceful degradation**: all core functionality continues if Qdrant is unavailable, with the API falling back to keyword-based search in FalkorDB.\n\n**Mermaid Diagram: Database Responsibilities**\n\nSources: [app.py75-78](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L75-L78) [app.py996-1500](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L996-L1500) [CLAUDE.md65-70](https://github.com/verygoodplugins/automem/blob/c2a4dae3/CLAUDE.md#L65-L70)\n\n* * *\n\n## FalkorDB (Graph Database)\n\n### Role and Capabilities\n\nFalkorDB is a Redis-compatible graph database that stores memories as nodes with typed relationships. It serves as the **authoritative data store** and enables:\n\n*   **Node storage**: Each memory is a `Memory` node with properties (content, type, importance, confidence, timestamp, tags)\n*   **Relationship traversal**: 11 relationship types (see [Relationship Types](/verygoodplugins/automem/3.2-relationship-types)) connect memories semantically\n*   **Keyword search**: Cypher queries perform content/tag matching\n*   **Temporal queries**: Filter memories by timestamp ranges\n*   **Pattern detection**: Store and query recurring patterns via enrichment metadata\n\n**Mermaid Diagram: FalkorDB Node and Edge Structure**\n\nSources: [app.py126-143](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L126-L143) [app.py2155-2220](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L2155-L2220)\n\n### Configuration\n\nEnvironment Variable\n\nDefault\n\nDescription\n\n`FALKORDB_HOST`\n\n`localhost`\n\nHostname or IP address\n\n`FALKORDB_PORT`\n\n`6379`\n\nRedis protocol port\n\n`FALKORDB_PASSWORD`\n\n_(none)_\n\nAuthentication password\n\n`FALKORDB_GRAPH`\n\n`memories`\n\nGraph database name\n\n**Connection Initialization**\n\nThe Flask app establishes connection at startup:\n\nSources: [app.py77-78](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L77-L78) [.env.example2-4](https://github.com/verygoodplugins/automem/blob/c2a4dae3/.env.example#L2-L4)\n\n### Persistence Configuration\n\nFalkorDB uses Redis AOF (Append-Only File) and RDB snapshots for durability. It provides:\n\n*   **Fast vector search**: Sub-100ms similarity queries over thousands of memories\n*   **Payload mirroring**: Stores memory content, tags, importance alongside vectors\n*   **Filtered search**: Combine vector similarity with tag/metadata filters\n*   **Batch operations**: Efficient bulk upserts for embedding generation\n\nQdrant is **optional** — if unavailable, AutoMem falls back to keyword-based search in FalkorDB.\n\n**Mermaid Diagram: Qdrant Collection Structure**\n\nSources: [app.py75-76](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L75-L76) [app.py924-994](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L924-L994)\n\n### Configuration\n\nEnvironment Variable\n\nDefault\n\nDescription\n\n`QDRANT_URL`\n\n_(none)_\n\nFull URL (e.g., `https://xyz.cloud.qdrant.io`)\n\n`QDRANT_API_KEY`\n\n_(none)_\n\nAPI key for authentication\n\n`QDRANT_COLLECTION`\n\n`memories`\n\nCollection name\n\n`VECTOR_SIZE`\n\n`3072`\n\nEmbedding dimensions (must match collection and provider)\n\n**Connection Initialization**\n\nQdrant client is initialized with error handling and dimension validation:\n\n**Dimension Validation**\n\nAutoMem validates vector dimensions before writes to prevent data corruption:\n\n**Migration Note**: Existing deployments using 768-dimensional embeddings should keep `VECTOR_SIZE=768` and `EMBEDDING_MODEL=text-embedding-3-small` until re-embedding via `scripts/reembed_embeddings.py`.\n\nSources: [app.py2120-2150](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L2120-L2150) [automem/utils/validation.py](https://github.com/verygoodplugins/automem/blob/c2a4dae3/automem/utils/validation.py) [.env.example5-8](https://github.com/verygoodplugins/automem/blob/c2a4dae3/.env.example#L5-L8) [CLAUDE.md151-153](https://github.com/verygoodplugins/automem/blob/c2a4dae3/CLAUDE.md#L151-L153)\n\n### Embedding Generation\n\nAutoMem uses a provider-based embedding system with automatic fallback.
