memories in AutoMem. These endpoints handle creating, retrieving, updating, and deleting memory records in both FalkorDB (graph database) and Qdrant (vector database).\n\nFor relationship management between memories, see [Relationship Operations](/verygoodplugins/automem/5.2-recall-operations). For background enrichment and embedding operations, see [Background Processing](/verygoodplugins/automem/4.3-background-processing). For administrative operations like bulk reprocessing, see [Admin Operations](/verygoodplugins/automem/5.4-consolidation-operations).\n\n* * *\n\n## Overview\n\nMemory operations provide the primary interface for storing and retrieving contextual information. All operations except `/health` require authentication via `AUTOMEM_API_TOKEN`. The system maintains dual storage: FalkorDB serves as the source of truth for graph data, while Qdrant provides semantic search capabilities.\n\n**Available Endpoints:**\n\nEndpoint\n\nMethod\n\nPurpose\n\nAuthentication\n\n`/memory`\n\nPOST\n\nCreate new memory\n\nAPI Token\n\n`/recall`\n\nGET\n\nSearch/retrieve memories\n\nAPI Token\n\n`/memory/<id>`\n\nPATCH\n\nUpdate existing memory\n\nAPI Token\n\n`/memory/<id>`\n\nDELETE\n\nRemove memory\n\nAPI Token\n\n`/memory/by-tag`\n\nGET\n\nFilter by tags\n\nAPI Token\n\nSources: [app.py72-73](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L72-L73) [app.py209-211](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L209-L211) [app.py1391-1401](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L1391-L1401)\n\n* * *\n\n## POST /memory - Creating Memories\n\nCreates a new memory node in FalkorDB and optionally stores its embedding in Qdrant. The operation executes synchronously for the primary write but queues background enrichment and embedding generation tasks.\n\n### Request Format\n\n**Required Fields:**\n\n*   `content` (string): Memory content, minimum 1 character\n\n**Optional Fields:**\n\n*   `type` (string): One of `Decision`, `Pattern`, `Preference`, `Style`, `Habit`, `Insight`, `Context` (default: auto-classified)\n*   `confidence` (float): 0.0-1.0, classification confidence (default: 0.9 if type provided)\n*   `tags` (array): Categorization tags, supports hierarchical syntax with `:` or `/` delimiters\n*   `importance` (float): 0.0-1.0, importance score (default: 0.5)\n*   `metadata` (object): Arbitrary JSON metadata\n*   `timestamp` (string): ISO 8601 timestamp (default: current UTC time)\n*   `embedding` (array): 768-dimensional vector (auto-generated if omitted)\n*   `id` (string): Custom UUID (auto-generated if omitted)\n*   `t_valid`, `t_invalid` (string): Temporal validity bounds\n*   `updated_at`, `last_accessed` (string): Tracking timestamps\n\nSources: [app.py119-122](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L119-L122) [app.py1842-1860](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L1842-L1860)\n\n### Memory Type Classification\n\nThe system uses `MemoryClassifier` to automatically determine memory type when not explicitly provided:\n\n**Classification Strategy:**\n\n1.  **Explicit Type**: If `type` parameter provided, use directly with `confidence=0.9`\n2.  **Regex Patterns**: Match content against predefined patterns for each memory type (fast, free)\n3.  **LLM Classification**: Use OpenAI GPT-4o-mini as fallback for complex content\n4.  **Default**: Assign `Context` type with low confidence (0.3) if all methods fail\n\nSources: [app.py996-1117](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L996-L1117) [app.py1862-1872](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L1862-L1872)\n\n### Data Flow\n\n**Processing Steps:**\n\n1.  **Validation**: Extract and validate required fields, normalize tags and timestamps\n2.  **Classification**: Determine memory type if not provided (regex → LLM → default)\n3.  **Tag Processing**: Compute hierarchical tag prefixes for efficient filtering\n4.  **Graph Write**: Execute `MERGE` operation in FalkorDB (immediate, blocking)\n5.  **Enrichment Queue**: Add to background queue for entity extraction and relationship building\n6.  **Embedding Handling**: Store provided embedding or queue for generation\n7.  **Response**: Return immediately with memory ID and enrichment status\n\nSources: [app.py1842-2019](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L1842-L2019) [app.py213-240](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L213-L240) [app.py280-329](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L280-L329)\n\n### Tag Processing\n\nTags support hierarchical structure using `:` or `/` delimiters. The system computes all prefixes for efficient filtering:\n\nThis enables prefix matching queries like `tags=slack` to match `slack:channel:general`, `slack:user:U123`, etc.\n\n**Implementation:**\n\n*   `_normalize_tag_list()`: Parse comma-separated or array tags\n*   `_expand_tag_prefixes()`: Split on `:` or `/` and generate cumulative prefixes\n*   `_compute_tag_prefixes()`: Deduplicate and lowercase all prefixes\n\nSources: [app.py280-329](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L280-L329) [app.py1897-1901](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L1897-L1901)\n\n### Response Format\n\nSources: [app.py1990-2019](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L1990-L2019)\n\n* * *\n\n## GET /recall - Retrieving Memories\n\nPerforms hybrid search combining vector similarity (Qdrant), keyword matching (FalkorDB), and tag filtering. Results are scored using a weighted multi-factor algorithm.\n\n### Query Parameters\n\nParameter\n\nType\n\nDescription\n\nDefault\n\n`query`\n\nstring\n\nFull-text search string\n\n\\-\n\n`embedding`\n\nstring\n\nComma-separated 768-d vector\n\n\\-\n\n`limit`\n\ninteger\n\nMax results (1-50)\n\n`10`\n\n`time_query`\n\nstring\n\nNatural language time expression\n\n\\-\n\n`start`\n\nstring\n\nISO timestamp lower bound\n\n\\-\n\n`end`\n\nstring\n\nISO timestamp upper bound\n\n\\-\n\n`tags`\n\nstring\\[\\]\n\nTag filters (multiple allowed)\n\n\\-\n\n`tag_mode`\n\nstring\n\n`any` or `all`\n\n`any`\n\n`tag_match`\n\nstring\n\n`prefix` or `exact`\n\n`prefix`\n\n**Example Requests:**\n\nSources: [app.py2020-2050](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L2020-L2050)\n\n### Hybrid Search Architecture\n\n**Search Strategy:**\n\n1.  **Vector Search** (`state.qdrant != None`): Semantic similarity using OpenAI embeddings\n2.  **Keyword Search** (FalkorDB): Content and tag matching using Cypher `CONTAINS`\n3.  **Trending Results** (no query): High-importance memories ordered by recency\n4.  **Deduplication**: Track seen IDs across sources\n5.  **Filtering**: Apply temporal and tag constraints\n6.  **Scoring**: Combine weighted factors into final score\n7.  **Relations**: Fetch connected memories (limited by `RECALL_RELATION_LIMIT`)\n\nSources: [app.py2051-2232](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L2051-L2232) [app.py924-994](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L924-L994) [app.py721-829](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L721-L829)\n\n### Scoring Algorithm\n\nThe final score combines multiple weighted components:\n\n**Component Calculations:**\n\n*   `vector_score`: Qdrant cosine similarity (0.0-1.0)\n*   `keyword_score`: Cypher query match score (keyword hits + phrase bonus)\n*   `tag_score`: `matched_tokens / total_query_tokens`\n*   `importance`: Direct value from memory (0.0-1.0)\n*   `confidence`: Type classification confidence (0.0-1.0)\n*   `recency_score`: `max(0, 1 - (age_days / 180))` (6-month decay)\n*   `exact_match`: 1.0 if exact phrase in metadata, else 0.0\n\nSources: [app.py200-207](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L200-L207) [app.py476-520](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L476-L520)\n\n### Keyword Search Implementation\n\nFalkorDB keyword search uses Cypher queries with dynamic scoring:\n\n**Scoring:**\n\n*   Content match: 2 points per keyword\n*   Tag match: 1 point per keyword\n*   Exact phrase bonus: +2 (content) or +1 (tag)\n\nSources: [app.py766-786](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L766-L786)\n\n### Tag Filtering\n\nSupports both prefix and exact matching with `any`/`all` modes:\n\n**Prefix Matching Example:**\n\n*   Query: `tags=slack`\n*   Matches: `slack:channel:general`, `slack:user:U123`, `slack:reaction:thumbs-up`\n*   Uses precomputed `tag_prefixes` field for O(1) lookup\n\n**Mode Behavior:**\n\n*   `any` (default): Match if ANY filter tag present\n*   `all`: Match only if ALL filter tags present\n\nSources: [app.py523-553](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L523-L553) [app.py831-862](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L831-L862)\n\n### Response Format\n\nSources: [app.py2196-2232](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L2196-L2232)\n\n* * *\n\n## PATCH /memory/:id - Updating Memories\n\nUpdates an existing memory node in FalkorDB and synchronizes changes to Qdrant. Content changes trigger automatic re-embedding.\n\n### Request Format\n\n**Updatable Fields:**\n\n*   `content`: Triggers re-embedding if changed\n*   `tags`: Recomputes `tag_prefixes` automatically\n*   `importance`, `confidence`, `type`: Update directly\n*   `metadata`: Merged with existing metadata (not replaced)\n*   `t_valid`, `t_invalid`: Update temporal bounds\n\n**Non-updatable Fields:**\n\n*   `id`: Immutable identifier\n*   `timestamp`: Preserved original creation time\n*   `updated_at`: Auto-set to current UTC time\n\nSources: [app.py2233-2258](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L2233-L2258)\n\n### Update Data Flow\n\n**Update Process:**\n\n1.  **Validation**: Verify memory exists (404 if not found)\n2.  **Field Processing**: Normalize tags, compute prefixes, validate types\n3.  **Graph Update**: Execute Cypher `SET` operation with changed fields\n4.  **Re-embedding**: Queue if `content` changed\n5.  **Qdrant Sync**: Update payload fields (non-blocking failure)\n6.  **Refresh**: Fetch updated node and return to client\n\nSources: [app.py2233-2344](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L2233-L2344)\n\n### Metadata Merge Behavior\n\nThe `metadata` field uses merge semantics, not replacement:\n\nTo remove a metadata field, explicitly set it to `null`.\n\nSources: [app.py2279-2284](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L2279-L2284)\n\n### Response Format\n\nSources: [app.py2337-2344](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L2337-L2344)\n\n* * *\n\n## DELETE /memory/:id - Deleting Memories\n\nRemoves a memory from both FalkorDB and Qdrant. The operation deletes the node, all its relationships, and the corresponding vector embedding.\n\n### Request Format\n\nNo request body required.\n\nSources: [app.py2345-2351](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L2345-L2351)\n\n### Deletion Data Flow\n\n**Deletion Process:**\n\n1.  **Graph Deletion**: Execute Cypher `DETACH DELETE` to remove node and relationships\n2.  **Vector Deletion**: Remove embedding from Qdrant (non-blocking failure)\n3.  **Response**: Confirm deletion success\n\nThe `DETACH DELETE` clause ensures all incoming and outgoing relationships are automatically removed, preventing orphaned edges.\n\nSources: [app.py2345-2408](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L2345-L2408)\n\n### Error Handling\n\nThe endpoint returns success even if the memory doesn't exist, following idempotent DELETE semantics:\n\nQdrant deletion failures are logged but don't fail the request, since FalkorDB is the source of truth.\n\nSources: [app.py2381-2408](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L2381-L2408)\n\n* * *\n\n## GET /memory/by-tag - Querying by Tags\n\nRetrieves memories filtered by tags, ordered by importance and recency. More performant than `/recall` when only tag filtering is needed.\n\n### Query Parameters\n\nParameter\n\nType\n\nDescription\n\nDefault\n\n`tags`\n\nstring\\[\\]\n\nTag filters (multiple values supported)\n\nRequired\n\n`limit`\n\ninteger\n\nMax results (1-100)\n\n`50`\n\n**Example Requests:**\n\nSources: [app.py2409-2421](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L2409-L2421)\n\n### Implementation\n\n**Query Strategy:**\n\n1.  **Vector-First**: If Qdrant available, use `scroll()` with tag filter\n2.  **Graph Fallback**: Query FalkorDB if Qdrant unavailable or returns no results\n3.  **Ordering**: Sort by `importance DESC, timestamp DESC`\n4.  **Relations**: Fetch connected memories for context\n5.  **Format**: Return in same format as `/recall` for consistency\n\nSources: [app.py2409-2466](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L2409-L2466) [app.py865-922](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L865-L922)\n\n### Cypher Query\n\nWhen using FalkorDB, the query leverages tag arrays:\n\nThis approach is more efficient than `/recall` for tag-only filtering because:\n\n*   No keyword extraction or scoring\n*   No vector search\n*   Direct index usage on `tags` property\n*   Simpler ordering logic\n\nSources: [app.py2438-2450](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L2438-L2450)\n\n### Response Format\n\nThe `score` field reflects the memory's `importance` when filtering by tags only.\n\nSources: [app.py2458-2466](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L2458-L2466)\n\n* * *\n\n## Authentication Flow\n\nAll memory operations require memories from AutoMem. The primary endpoint is `GET /recall`, which implements hybrid search combining semantic vectors, keyword matching, graph relationships, and temporal signals.\n\nFor memory creation and updates, see [Memory Operations](/verygoodplugins/automem/5.1-memory-operations). For graph relationship traversal, see [Relationship Operations](/verygoodplugins/automem/5.3-relationship-operations).\n\n## Endpoint Overview\n\nThe recall system provides a single unified endpoint that supports multiple search strategies:\n\n    GET /recall\n    \n\nThis endpoint combines nine scoring components into a hybrid ranking system and supports both basic retrieval and advanced graph expansion. Authentication is required via `Authorization: Bearer <token>` header or `X-API-Key` header.\n\n**Sources:** [app.py1600-2200](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L1600-L2200) [docs/API.md42-84](https://github.com/verygoodplugins/automem/blob/c2a4dae3/docs/API.md#L42-L84)\n\n## Request Parameters\n\n### Basic Search Parameters\n\nParameter\n\nType\n\nDefault\n\nDescription\n\n`query`\n\nstring\n\n\\-\n\nNatural language search query or wildcard `*`\n\n`embedding`\n\narray\\[float\\]\n\n\\-\n\nPre-computed embedding vector for semantic search\n\n`limit`\n\ninteger\n\n10\n\nMaximum results to return (capped at `RECALL_MAX_LIMIT=100`)\n\n`sort` / `order_by`\n\nstring\n\n`score`\n\nSort mode: `score`, `time_desc`, `time_asc`, `updated_desc`, `updated_asc`\n\nThe `query` parameter triggers keyword extraction and full-text search via [app.py442-550](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L442-L550) When `query=\"*\"` or is omitted, the system returns trending memories ordered by importance.\n\n**Sources:** [app.py206-207](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L206-L207) [app.py378-439](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L378-L439) [tests/test\\_api\\_endpoints.py378-419](https://github.com/verygoodplugins/automem/blob/c2a4dae3/tests/test_api_endpoints.py#L378-L419)\n\n### Tag Filtering Parameters\n\nParameter\n\nType\n\nDefault\n\nDescription\n\n`tags`\n\narray\\[string\\]\n\n\\-\n\nTags to filter by (can specify multiple)\n\n`exclude_tags`\n\narray\\[string\\]\n\n\\-\n\nTags to exclude (supports prefix matching)\n\n`tag_mode`\n\nstring\n\n`any`\n\nMatch mode: `any` or `all`\n\n`tag_match`\n\nstring\n\n`prefix`\n\nMatch type: `prefix` or `exact`\n\nTag filtering uses pre-computed `tag_prefixes` stored in both FalkorDB and Qdrant for efficient prefix matching. The `exclude_tags` parameter removes any memory containing ANY of the specified tags.\n\n**Sources:** [app.py246-348](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L246-L348) [automem/utils/tags.py](https://github.com/verygoodplugins/automem/blob/c2a4dae3/automem/utils/tags.py) [docs/API.md46-83](https://github.com/verygoodplugins/automem/blob/c2a4dae3/docs/API.md#L46-L83)\n\n### Temporal Filtering Parameters\n\nParameter\n\nType\n\nDefault\n\nDescription\n\n`time_query`\n\nstring\n\n\\-\n\nNatural language time expression (e.g., \"last week\")\n\n`start`\n\nstring\n\n\\-\n\nISO 8601 start timestamp\n\n`end`\n\nstring\n\n\\-\n\nISO 8601 end timestamp\n\nTime queries are parsed by `_parse_time_expression()` from [automem/utils/time.py](https://github.com/verygoodplugins/automem/blob/c2a4dae3/automem/utils/time.py) which supports expressions like \"last 7 days\", \"yesterday\", \"last month\". Explicit timestamps override natural language queries.\n\n**Sources:** [automem/utils/time.py](https://github.com/verygoodplugins/automem/blob/c2a4dae3/automem/utils/time.py) [app.py404-409](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L404-L409) [tests/test\\_api\\_endpoints.py412-507](https://github.com/verygoodplugins/automem/blob/c2a4dae3/tests/test_api_endpoints.py#L412-L507)\n\n### Context Hint Parameters\n\nParameter\n\nType\n\nDefault\n\nDescription\n\n`context`\n\nstring\n\n\\-\n\nContextual hint for ranking (e.g., programming task)\n\n`language`\n\nstring\n\n\\-\n\nProgramming language context (e.g., \"python\")\n\n`active_path`\n\nstring\n\n\\-\n\nFile path being edited (auto-detects language)\n\n`context_tags`\n\narray\\[string\\]\n\n\\-\n\nTags to boost in scoring\n\n`context_types`\n\narray\\[string\\]\n\n\\-\n\nMemory types to prioritize\n\n`priority_ids`\n\narray\\[string\\]\n\n\\-\n\nSpecific memory IDs to guarantee inclusion\n\nContext hints influence the 9-component scoring system but do not filter results. Expansion filtering parameters only apply to expanded memories, never to seed results.\n\n**Sources:** [app.py156-157](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L156-L157) combines signals from [automem/utils/scoring.py](https://github.com/verygoodplugins/automem/blob/c2a4dae3/automem/utils/scoring.py) using weights defined in [automem/config.py159-165](https://github.com/verygoodplugins/automem/blob/c2a4dae3/automem/config.py#L159-L165):\n\n    final_score = (vector_score × SEARCH_WEIGHT_VECTOR) +\n                  (keyword_score × SEARCH_WEIGHT_KEYWORD) +\n                  (exact_match_score × SEARCH_WEIGHT_EXACT) +\n                  (importance × SEARCH_WEIGHT_IMPORTANCE) +\n                  (confidence × SEARCH_WEIGHT_CONFIDENCE) +\n                  (recency_score × SEARCH_WEIGHT_RECENCY) +\n                  (tag_score × SEARCH_WEIGHT_TAG) +\n                  relation_score +\n                  temporal_score\n    \n\n**Default Weight Configuration:**\n\nComponent\n\nEnvironment Variable\n\nDefault Value\n\nVector\n\n`SEARCH_WEIGHT_VECTOR`\n\n0.25\n\nKeyword\n\n`SEARCH_WEIGHT_KEYWORD`\n\n0.15\n\nExact\n\n`SEARCH_WEIGHT_EXACT`\n\n0.25\n\nImportance\n\n`SEARCH_WEIGHT_IMPORTANCE`\n\n0.05\n\nConfidence\n\n`SEARCH_WEIGHT_CONFIDENCE`\n\n0.05\n\nRecency\n\n`SEARCH_WEIGHT_RECENCY`\n\n0.10\n\nTag\n\n`SEARCH_WEIGHT_TAG`\n\n0.10\n\n**Sources:** [automem/config.py159-165](https://github.com/verygoodplugins/automem/blob/c2a4dae3/automem/config.py#L159-L165) [app.py612-682](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L612-L682) using Qdrant when available:\n\nWhen Qdrant is unavailable, the system gracefully degrades to graph-only mode using keyword and metadata scoring.\n\n**Sources:** [app.py612-682](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L612-L682) [app.py1850-1950](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L1850-L1950)\n\n#### Keyword Search Component\n\nImplemented in `_graph_keyword_search()` at [app.py442-550](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L442-L550) using Cypher queries:\n\n*   Extracts keywords via `_extract_keywords()` from [automem/utils/text.py](https://github.com/verygoodplugins/automem/blob/c2a4dae3/automem/utils/text.py)\n*   Matches against `m.content` and `m.tags` fields in FalkorDB\n*   Scores based on keyword frequency and phrase containment\n*   Returns memories ordered by `score DESC, m.importance DESC, m.timestamp DESC`\n\n**Sources:** [app.py442-550](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L442-L550) [app.py224-240](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L224-L240) `_parse_metadata_field()` from [automem/utils/scoring.py](https://github.com/verygoodplugins/automem/blob/c2a4dae3/automem/utils/scoring.py):\n\n*   **Importance**: Direct multiplication by weight (0.0-1.0 range)\n*   **Confidence**: Classification confidence from memory type detection\n*   **Recency**: Exponential decay based on time since last access\n*   **Tag**: Overlap ratio between query tags and memory tags\n\n**Sources:** [automem/utils/scoring.py](https://github.com/verygoodplugins/automem/blob/c2a4dae3/automem/utils/scoring.py) [app.py175-176](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L175-L176)\n\n## Graph Expansion\n\n### Relationship Expansion Flow\n\nThe `_expand_related_memories()` function from [automem/api/recall.py](https://github.com/verygoodplugins/automem/blob/c2a4dae3/automem/api/recall.py) implements multi-hop graph traversal. It uses Cypher `MATCH` patterns to follow typed relationship edges (e.g., `RELATES_TO`, `PREFERS_OVER`, `LEADS_TO`).\n\n**Key Configuration Constants:**\n\n*   `RECALL_RELATION_LIMIT` (default: 5) - Max edges per seed memory\n*   `RECALL_EXPANSION_LIMIT` (default: 25) - Total expanded memories cap\n\n**Sources:** [automem/api/recall.py](https://github.com/verygoodplugins/automem/blob/c2a4dae3/automem/api/recall.py) [automem/config.py156-157](https://github.com/verygoodplugins/automem/blob/c2a4dae3/automem/config.py#L156-L157) [tests/test\\_api\\_endpoints.py18](https://github.com/verygoodplugins/automem/blob/c2a4dae3/tests/test_api_endpoints.py#L18-L18)\n\n### Entity Expansion Flow\n\nWhen `expand_entities=true`, the system performs multi-hop reasoning via entity tags:\n\n1.  Extract entity tags from query (format: `entity:<type>:<slug>`)\n2.  Find memories with matching entity tags (first hop)\n3.  Extract entities from matched memories\n4.  Find memories containing those entities (second hop)\n\nExample: Query \"What is Sarah's sister's job?\" → Find \"Sarah\" entity → Find Sarah's relationships → Find \"sister Rachel\" → Find Rachel's job.\n\n**Sources:** [docs/API.md62](https://github.com/verygoodplugins/automem/blob/c2a4dae3/docs/API.md#L62-L62) Common failure modes:\n\nError\n\nCause\n\nBehavior\n\nOpenAI API rate limit\n\nExceeded quota\n\nRetries with exponential backoff (handled by OpenAI SDK)\n\nMissing memory content\n\nDeleted between enumeration and fetch\n\nLogged, skipped, processing continues\n\nQdrant connection failure\n\nNetwork issue or Qdrant down\n\nLogged, FalkorDB still updated (graceful degradation)\n\nInvalid content format\n\nNull or non-string content\n\nLogged, skipped\n\nAll errors are logged with structured context:\n\n    logger.exception(\"Failed to generate embeddings for batch\", extra={\"batch_ids\": ids})\n    \n\n**Sources:** [app.py2211-2245](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L2211-L2245)\n\n* * *\n\n## Security Model\n\n### Threat Model\n\nAdmin operations can:\n\n*   Force expensive OpenAI API calls (re-embedding entire database)\n*   Trigger resource-intensive enrichment reprocessing\n*   Access operational metrics (enrichment statistics)\n\n**Without admin token protection**, a compromised API token could:\n\n1. Practices\n\nPractice\n\nRationale\n\nImplementation\n\n**Separate tokens**\n\nLimits blast radius of API token compromise\n\nUse different values for `AUTOMEM_API_TOKEN` and `ADMIN_API_TOKEN`\n\n**Rotate periodically**\n\nReduces window of exposure\n\nRegenerate tokens monthly, update all clients\n\n**Restrict admin access**\n\nMinimize privilege escalation risk\n\nShare admin token only with operations team\n\n**Use headers, not query params**\n\nPrevents token leakage in logs\n\nPrefer `Authorization: Bearer` and `X-Admin-Token` headers\n\n**Monitor admin operations**\n\nDetect anomalous usage\n\nAlert on high-frequency `/admin/reembed` calls\n\n**Audit admin calls**\n\nForensic capability\n\nLog admin operations with IP, timestamp, token hash\n\n**Sources:** [app.py1377-1388](https://github.com/verygoodplugins/automem/blob/c2a4dae3/app.py#L1377-L1388)
