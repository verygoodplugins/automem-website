For setup instructions, see [Getting Started](/verygoodplugins/mcp-automem/2-getting-started).\n\n## What is mcp-automem\n\n**mcp-automem** is a Node.js package that provides memory capabilities to AI assistants via the Model Context Protocol. It does **not** store memories itself—it acts as a translation layer between AI platforms and a separate AutoMem service that handles storage.\n\n**Package Identity:**\n\n*   npm: `@verygoodplugins/mcp-automem` [package.json2-3](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/package.json#L2-L3)\n*   Version: 0.12.0 [package.json3](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/package.json#L3-L3)\n*   Main entry: `dist/index.js` [package.json5](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/package.json#L5-L5)\n*   Binary: `mcp-automem` [package.json11-13](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/package.json#L11-L13)\n\n### System Position Diagram\n\n**Sources:** Diagram 1 from high-level system diagrams, [package.json1-83](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/package.json#L1-L83) [README.md260-281](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/README.md#L260-L281)\n\n## The Bridge Pattern\n\nmcp-automem implements a **bridge pattern**: it translates MCP protocol calls from AI platforms into HTTP API requests to the AutoMem service. This separation allows:\n\n1.  **Single backend**: Multiple AI platforms share one memory store\n2.  **Platform independence**: The memory service doesn't need MCP knowledge\n3.  **Flexible deployment**: MCP client and memory service can run on different machines\n\nComponent\n\nRepository\n\nRole\n\n**mcp-automem** (this You have three deployment options:\n\n**Local Development** (fastest setup):\n\nService runs at `http://localhost:8001` with no authentication.\n\n**Railway Cloud** (recommended for production): Deploy via one-click Railway button, receives auto-generated API key and public URL.\n\n**Self-Hosted**: Use Docker Compose, Kubernetes, or any container platform.\n\nSee the [AutoMem Installation Guide](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/AutoMem Installation Guide) for complete deployment instructions.\n\n**Sources:** [INSTALLATION.md1-57](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/INSTALLATION.md#L1-L57) [README.md105-125](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/README.md#L105-L125)\n\n* * *\n\n## Installation Flow\n\nThe setup process follows a two-phase pattern: deploy backend, then connect clients.\n\nThe `npx @verygoodplugins/mcp-automem setup` command is defined in [package.json12](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/package.json#L12-L12) as the `mcp-automem` binary, which routes to [src/index.ts](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/src/index.ts) in CLI mode when invoked with the `setup` argument.\n\n**Sources:** [INSTALLATION.md170-194](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/INSTALLATION.md#L170-L194) [README.md148-163](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/README.md#L148-L163)\n\n* * *\n\n## Setup Wizard\n\n### Running the Setup Wizard\n\nThe setup wizard performs four operations:\n\n1.  **Prompts for configuration** - Endpoint and API key\n2.  **Validates connectivity** - Tests `/health` endpoint via `AutoMemClient`\n3.  **Creates `.env` file** - Stores configuration in current directory\n4.  **Prints config snippets** - Platform-specific MCP server configurations\n\n### Configuration File Locations\n\nThe setup wizard creates `.env` in the **current directory** where you run the command. When the MCP server starts, it uses `dotenv` to load configuration from `.env` if present - see [package.json55](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/package.json#L55-L55) for the `dotenv` dependency.\n\n**Key configuration variables:**\n\n*   `AUTOMEM_ENDPOINT` (required) - URL to AutoMem service\n*   `AUTOMEM_API_KEY` (optional) - API key for authenticated instances\n\n**Sources:** [INSTALLATION.md170-194](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/INSTALLATION.md#L170-L194) [package.json53-56](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/package.json#L53-L56)\n\n* * *\n\n## MCP Server Entry Points\n\nThe mcp-automem package operates in two distinct modes based on command-line arguments:\n\n**Server Mode:** When AI platforms (Claude Desktop, Cursor, etc.) launch the MCP server, they invoke `npx -y @verygoodplugins/mcp-automem` with no arguments. This starts `StdioServerTransport` which listens on stdin/stdout for JSON-RPC messages.\n\n**CLI Mode:** When users run `npx @verygoodplugins/mcp-automem setup`, the presence of arguments triggers CLI mode, routing to command handlers in `src/cli/commands/`.\n\nBoth modes use the same `AutoMemClient` class ([src/automem-client.ts](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/src/automem-client.ts)) for HTTP communication with the AutoMem service.\n\n**Sources:** [src/index.ts1-20](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/src/index.ts#L1-L20) [package.json11-13](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/package.json#L11-L13)\n\n* * *\n\n## Platform Configuration Files\n\nDifferent AI platforms store MCP server configuration in different locations:\n\nThe setup wizard prints the appropriate configuration snippet for each platform. Setup wizard reports \"Connection failed\" when testing endpoint.\n\n**Causes and solutions:**\n\n1.  **AutoMem service not running**\n    \n2.  **Incorrect endpoint URL**\n    \n    *   Local: Must be `http://localhost:8001` (not `127.0.0.1` in some setups)\n    *   Railway: Must include `https://` scheme\n    *   Check for trailing slashes (should be omitted)\n3.  **Firewall blocking connection**\n    \n    *   Check if port 8001 is accessible\n    *   Verify Railway service public networking is enabled\n\n**Sources:** [INSTALLATION.md1189-1216](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/INSTALLATION.md#L1189-L1216)\n\n### MCP Server Not Appearing in AI Platform\n\n**Symptom:** After configuration, the memory tools don't appear in Claude Desktop/Cursor.\n\n**Solutions by platform:**\n\n**Claude Desktop:**\n\n*   Restart Claude Desktop completely (quit, not just window close)\n*   Verify JSON syntax in `claude_desktop_config.json` (no trailing commas)\n*   Check file location matches your OS (see [Platform Configuration Files](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/Platform Configuration Files))\n\n**Cursor:**\n\n*   Reload Cursor window\n*   Verify `~/.cursor/mcp.json` has correct `command` and `env` fields\n*   Check `.cursor/rules/automem.mdc` has valid YAML frontmatter\n\n**Sources:** [INSTALLATION.md1219-1234](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/INSTALLATION.md#L1219-L1234)\n\n### Authentication Errors\n\n**Symptom:** \"Unauthorized\" or \"403 Forbidden\" errors when using memory tools.\n\n**Causes and solutions:**\n\n1.  **Missing API key for Railway deployment**\n    \n    *   Railway deployments require `AUTOMEM_API_KEY`\n    *   Local development (localhost) does not require API key\n    *   Add/update key in `.env` or platform config\n2.  **Incorrect API key format**\n    \n    *   Key should be provided as-is (no \"Bearer\" prefix in config)\n    *   `AutoMemClient` automatically adds \"Bearer\" prefix to Authorization header\n\n**Sources:** [INSTALLATION.md1198-1202](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/INSTALLATION.md#L1198-L1202) [src/automem-client.ts15-28](https://github.com/verygoodplugins/mcp-automem/blob/7e26148d/src/automem-client.ts#L15-L28)\n\n### Memory Not Stored\n\n**Symptom:** `store_memory` succeeds but memory not recalled later.\n\n**Debug steps:**\n\n1.  **Check database health**\n    \n    Both FalkorDB and Qdrant should show as connected.\n    \n2.  **Verify memory was written**\n    \n3.  **Check content length**\n    \n    *   Memories over 2000 characters are rejected\n    *   Memories over 500 characters may be summarized by backend\n\n**Sources:** `beforeShellExecution`, and `stop` hooks.\n\n**Removed Because**:\n\n*   Hook system proved unreliable\n*   Overly complex (11 template files, 563 lines of hook scripts)\n*   `.mdc` rule files
